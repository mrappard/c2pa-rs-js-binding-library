# C2PA Rust JavaScript Library

`c2pa-rs-javascript-library` is a TypeScript library for signing and verifying C2PA Content Credentials from JavaScript. For the current structured-text testing work, it uses the temporary Rust implementation in [`mrappard/c2pa-rs-text-support`](https://github.com/mrappard/c2pa-rs-text-support) behind a WebAssembly boundary, giving JavaScript consumers a small API over the underlying Rust signing and verification logic.

## What It Does

The library currently exposes:

- `signAsset(...)` to sign an asset and produce a signed asset plus generated manifest bytes
- `verifyAsset(...)` to inspect an asset and return verification state plus recognized manifest data
- Type exports for supported formats, signing algorithms, manifests, thumbnails, ingredients, and verification results

The Rust layer handles the C2PA-specific work, including manifest parsing, signing, and verification. Once the structured-text changes are validated, this wording can be switched back to the upstream `c2pa-rs` project.

## Supported Formats

The current implementation supports these MIME types:

- `application/pdf`
- `image/jpeg`
- `image/png`
- `image/svg+xml`
- `image/x-adobe-dng`

## Project Structure

- `crate/`: Rust source compiled to WebAssembly
- `src/`: TypeScript wrapper API
- `pkg/`: output generated by `wasm-pack`
- `dist/`: compiled TypeScript output


## API Overview

### Verify an Asset

```ts
import { verifyAsset } from 'c2pa-rs-javascript-library';

const assetBytes = new Uint8Array(/* asset bytes */);
const trustedCertificates = [
  `-----BEGIN CERTIFICATE-----
  ...
  -----END CERTIFICATE-----`,
];

const result = await verifyAsset(
  'image/png',
  assetBytes,
  trustedCertificates
);

console.log(result.state);
console.log(result.manifests);
```

### Sign an Asset

```ts
import { signAsset } from 'c2pa-rs-javascript-library';

const assetBytes = new Uint8Array(/* asset bytes */);
const manifestDefinition = {
  claim_generator: 'c2pa-rs-javascript-library',
  title: 'Example Asset',
  assertions: [],
};

const signCert = new Uint8Array(/* signing certificate bytes */);
const privateKey = new Uint8Array(/* private key bytes */);

const result = await signAsset(
  'image/png',
  assetBytes,
  manifestDefinition,
  signCert,
  privateKey,
  'es256'
);

console.log(result.signedAsset);
console.log(result.manifest);
```

## Scripts

- `npm run build:wasm`: build the Rust/WASM package for bundlers
- `npm run test:wasm`: build the Rust/WASM package for Node.js tests
- `npm run build:ts`: compile TypeScript into `dist/`
- `npm run build`: build both WASM and TypeScript outputs
- `npm test`: build the Node.js WASM target and run the test suite

## Notes

- The TypeScript API is currently a thin wrapper over the generated WASM bindings.
- The repository still contains some early scaffold artifacts in code and metadata, but the implemented direction is a C2PA signing and verification library rather than a generic Rust/WASM example.

## npm Publish Checklist

Before publishing:

1. Run `npm test`
2. Run `npm pack --dry-run`
3. Confirm the tarball includes `dist/` plus the generated WASM files
4. Bump the version in `package.json`
5. Log into npm with `npm login`

Publish:

```bash
npm publish
```

For a scoped public package:

```bash
npm publish --access public
```

The `prepack` script automatically rebuilds and stages the package into `dist/` before publish.
